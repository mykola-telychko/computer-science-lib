### Unique Paths II

Цей документ пояснює підхід до розв'язання задачі "Unique Paths II", яка передбачає пошук кількості унікальних шляхів у сітці з перешкодами.

#### **Опис методу**

Для розв'язання цієї задачі використовується метод динамічного програмування (Dynamic Programming). Ми модифікуємо вхідну сітку (grid) для зберігання кількості унікальних шляхів до кожної клітинки.

1.  **Обробка граничних випадків**:
    * Якщо початкова клітинка `(0, 0)` є перешкодою (тобто `obstacleGrid[0][0] === 1`), шляхів не існує. У цьому випадку повертаємо `0`.
    * Якщо `(0, 0)` не є перешкодою, встановлюємо `obstacleGrid[0][0] = 1`, оскільки існує лише один спосіб потрапити в цю точку (залишитися на місці).

2.  **Ініціалізація першого рядка та стовпця**:
    * Для **першого рядка** (`i = 0`): кількість шляхів до клітинки `(0, j)` залежить від клітинки зліва `(0, j-1)`. Якщо поточна клітинка не є перешкодою і до попередньої можна дістатися, то `obstacleGrid[0][j] = obstacleGrid[0][j-1]`. В іншому випадку, `obstacleGrid[0][j] = 0`.
    * Аналогічно, для **першого стовпця** (`j = 0`): кількість шляхів до клітинки `(i, 0)` залежить від клітинки зверху `(i-1, 0)`. Якщо поточна клітинка не є перешкодою і до клітинки зверху можна дістатися, то `obstacleGrid[i][0] = obstacleGrid[i-1][0]`. Інакше, `obstacleGrid[i][0] = 0`.

3.  **Заповнення таблиці DP**:
    * Для кожної клітинки `(i, j)` (де `i >= 1` і `j >= 1`):
        * Якщо клітинка не є перешкодою (`obstacleGrid[i][j] === 0`), кількість унікальних шляхів до неї є сумою шляхів із клітинки зверху (`obstacleGrid[i-1][j]`) та зліва (`obstacleGrid[i][j-1]`).
        * Якщо клітинка є перешкодою (`obstacleGrid[i][j] === 1`), встановлюємо `obstacleGrid[i][j] = 0`, оскільки через неї неможливо пройти.

4.  **Результат**:
    * Кінцеве значення у клітинці `obstacleGrid[m-1][n-1]` (нижній правий кут) є загальною кількістю унікальних шляхів.

#### **Складність алгоритму**

* **Часова складність**: `O(m * n)`, де `m` — кількість рядків, а `n` — кількість стовпців. Ми проходимо по кожній клітинці сітки рівно один раз.
* **Просторова складність**: `O(1)`. Рішення використовує вхідну сітку для зберігання результатів, не потребуючи додаткової пам'яті.

#### **Приклади**

**Приклад 1:**

* **Вхід**: `obstacleGrid = [[0,0,0],[0,1,0],[0,0,0]]`
* **Результат**: `2`
* **Пояснення**:
    * **Ініціалізація**: `obstacleGrid[0][0]` стає `1`.
    * **Заповнення**:
        * Перший рядок: `[1, 1, 1]`.
        * Перший стовпець: `[1, 1, 1]`.
        * Клітинка `(1,1)` є перешкодою, тому `obstacleGrid[1][1] = 0`.
        * Клітинка `(1,2)`: `obstacleGrid[1][2] = obstacleGrid[0][2] + obstacleGrid[1][1] = 1 + 0 = 1`.
        * Клітинка `(2,1)`: `obstacleGrid[2][1] = obstacleGrid[1][1] + obstacleGrid[2][0] = 0 + 1 = 1`.
        * Клітинка `(2,2)`: `obstacleGrid[2][2] = obstacleGrid[1][2] + obstacleGrid[2][1] = 1 + 1 = 2`.
    * **Вихід**: `obstacleGrid[2][2]` дорівнює `2`.

**Приклад 2:**

* **Вхід**: `obstacleGrid = [[0,1],[0,0]]`
* **Результат**: `1`
* **Пояснення**:
    * **Ініціалізація**: `obstacleGrid[0][0]` стає `1`.
    * **Заповнення**:
        * Перший рядок: `[1, 0]` (друга клітинка — перешкода).
        * Перший стовпець: `[1, 1]`.
        * Клітинка `(1,1)`: `obstacleGrid[1][1] = obstacleGrid[0][1] + obstacleGrid[1][0] = 0 + 1 = 1`.
    * **Вихід**: `obstacleGrid[1][1]` дорівнює `1`.
